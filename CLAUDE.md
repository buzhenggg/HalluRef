## 开发工作流（必须严格遵守）

### 功能实现流程
每次实现一个新功能时，必须按以下顺序执行：
1. 先更新 `docs/api.md` 中的接口文档，记录新接口的路径、方法、参数、返回值
2. 在 `tests/` 目录下编写对应的测试文件
3. 实现功能代码
4. 运行 `npm run test` 确保所有测试通过
5. 只有测试全部通过后才能认为任务完成

### 代码修改规则
每次修改代码后，必须运行 `npm run test`，确保所有测试通过。
如果测试失败，尝试调试，调试失败恢复代码并向我报告错误。



**环境请使用 conda 创建虚拟环境**，虚拟环境名字叫 ‘halluref’
